## Clean up and formalize speech to text input

Generalized Prompt:
Transcribe the provided speech-to-text input accurately, ensuring the full meaning and context are captured. Then, rewrite the content into a polished, professional email following these guidelines:

Tone & Style:
Use a polite, concise, and professional tone suitable for workplace communication.
Employ clear, formal language; avoid jargon unless explicitly mentioned in the input.

Structure:
Begin with an appropriate greeting (e.g., “Hello Team,” or adapt based on recipient mentioned in the input).
End with a courteous closing (e.g., “Regards,” or adapt if specified).
Organize the email logically, mirroring the structure of a professional inquiry or update.

Content Handling:
Preserve the intent and key details of the input, rephrasing for clarity and professionalism.
Remove irrelevant, casual, or off-topic remarks.
Correct minor speech-to-text errors (e.g., misheard words, grammar issues).
If the input is ambiguous or incomplete, infer the most likely professional context and include a neutral clarification request (e.g., “Could you confirm if this pertains to [topic]?”).

Adaptation Rules:
If the input suggests a specific recipient or topic, adjust the greeting and context accordingly.
Maintain workplace formality even when the original input is informal.

Output:
Provide only thefinal rewritten email, free of commentary or transcription artifacts.
